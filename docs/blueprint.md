# **App Name**: AttendEase

## Core Features:

- Role-Based Authentication: Implement a secure login system with distinct roles for users and admins. Stores roles in backend. Allows different access and privileges based on roles.
- User Attendance Punch: Allow users to punch in and out, recording their attendance with timestamps. Uses the Google Sheets backend for record storage.
- Attendance Visualization: Display attendance records for each user. Enable filtering by date. All the records will be visible to admin and the personal records visible to the user.
- Leave Application: Users can submit leave requests via the app. Leave details include date and time.
- Admin Dashboard: Provide an admin dashboard for viewing attendance records of all users. Enables filtering and reporting functionality.
- Leave Approval: Admins can approve or reject user leave requests.
- Report Generation: Admins can generate attendance reports in CSV or PDF format.

## Style Guidelines:

- Primary color: Deep blue (#3F51B5) to convey trust and professionalism.
- Background color: Very light blue (#E8EAF6), almost white, for a clean and modern feel.
- Accent color: Orange (#FF9800) to highlight key actions and calls to action.
- Headline font: 'Inter' (sans-serif) for a modern, neutral look.
- Body font: 'PT Sans' (sans-serif) for readability and warmth. To be used for the longer body of text.
- Use consistent and clear icons from Material Design for actions and status indicators.
- Maintain a clean and organized layout with a clear information hierarchy. Prioritize key data and actions.
- Incorporate subtle animations for feedback on actions, such as button presses and data updates.
